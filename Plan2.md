Medical Study Search Application

Recommended Tech Stack

Backend





Python + FastAPI — minimal boilerplate, async support, auto-generates API docs



PyMuPDF (fitz) — fastest PDF text extraction library



python-docx — Word document text extraction



SQLite FTS5 — built-in full-text search, no external service needed, good ranking



pydicom — read DICOM metadata and pixel data on the backend

Frontend





React + Vite + TypeScript — fast dev server, lightweight



Tailwind CSS — utility-first, no design setup overhead



Cornerstone.js + cornerstoneWADOImageLoader — battle-tested DICOM renderer in browser

Stage 1 — Document Search

Build-time Indexer (scripts/build_index.py)

This is a standalone script run once before starting the app, or whenever documents change. It is not part of the running server.

# Run at build time:
python scripts/build_index.py

What it does:





Reads DOCS_ROOT and STUDY_ID_REGEX from config.py



Walks all subdirectories, finds every .pdf and .docx file



Extracts full text (PyMuPDF for PDF, python-docx for DOCX)



Extracts Study ID from text using the configured regex (e.g. r'Study\s*ID[:\s]+(\S+)')



Drops and rebuilds a SQLite FTS5 virtual table



Inserts (study_id, source_dir, filename, full_text) rows



Writes data/index.db — the backend reads from this file at runtime

Runtime server (backend/)

The FastAPI server is read-only against the index. It never writes to index.db.

API endpoints (Stage 1)





GET /api/search?q={query} — runs FTS5 BM25-ranked query, returns [{study_id, filename, snippet}]

Stage 2 — DICOM Viewer (after Stage 1 is working)

DICOM directory structure assumption

dicom_root/
├── STUDY_001/
│   ├── subject_01/    ← up to 5 subject datasets per study
│   ├── subject_02/
│   └── ...
├── STUDY_002/
│   └── ...

Backend (dicom_service.py)





GET /api/study/{study_id}/dicoms — lists up to 5 subject subdirectories, returns file paths



GET /api/dicom/{study_id}/{subject}/{filename} — streams the raw DICOM file bytes

Frontend (Stage 2)





Clicking a study card opens a side panel / modal



Cornerstone.js loads each DICOM file via the streaming endpoint and renders it



Show up to 5 subject viewer panels



Key Notes





Indexer is a one-off script — run python scripts/build_index.py once at setup; no cron needed for a hackathon (but trivially wrappable in one if required later)



Study ID regex is configurable in config.py — adjust once you add real documents to confirm the exact pattern



index.db is static at runtime — the server opens it read-only; no locking or write concerns



No auth — internal tool; can add later



No Docker — direct uvicorn + vite dev for hackathon speed
